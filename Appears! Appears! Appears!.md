## 1. 컨셉과 훅

### 1.1 한 줄 컨셉
전투·대화·길찾기에 시간이 드는 JRPG를, 현대인이 빠르게 즐길 수 있도록 인크리멘탈 문법으로 압축한다. JRPG의 모든 요소(전투창, 마을, 여관, 교회, 카지노, 대장간, 항아리, 보물상자, 동료, 장비, 도감, 회차)가 존재하되, 전부 해금·업그레이드의 대상이 된다.

### 1.2 핵심 훅 ① — 멀티 전투창
- 드퀘식 1인칭 전투창(검은 배경, 흰 테두리, 몬스터 정면 도트, 텍스트 로그)이 화면 위에 **여러 개 동시에** 뜬다.
- 필드의 파티는 전투창이 열려도 멈추지 않는다. 몬스터를 들이받을 때마다 창이 "발사"되고, 창은 알아서 싸운다.
- 논리적 근거: 드퀘 1인칭 시점에서는 아군이 화면에 보이지 않는다 = 파티는 카메라 뒤에 있다 = **모든 창의 카메라 뒤에 동시에 존재해도 모순이 없다** (파티 편재의 원리). 3인칭 사이드뷰였다면 불가능한 트릭.
- 후반 화면이 전투창으로 도배되는 과잉이 곧 인크리멘탈의 쾌감이자 장르 풍자("JRPG는 어차피 전투창만 본다"), 그리고 대표 스크린샷이다.

### 1.3 핵심 훅 ② — 자라나는 마을 (눈에 보이는 업그레이드 트리)
- 업그레이드 = 텍스트 리스트가 아니라, 빈 터였던 마을에 건물이 물리적으로 솟아나는 것.
- 건물 해금 연출 = NPC가 걸어 들어와 짐을 풀고 건물이 세워짐 (저항군 합류, §2 참조).
- 초반 허허벌판 vs 후반 번화한 마을 + 전투창 도배 = 성장 서사가 스크린샷 한 장에 읽힌다.

### 1.4 핵심 훅 ③ — 상시 설명창 (검시 텍스트)
- 커서가 닿는 모든 것의 플레이버 텍스트가 화면 하단 고정 대화창에 흐른다. 고전 어드벤처 + 언더테일 감성. 튜토리얼을 전량 대체하는 채널 (§3.4).

### 1.5 플레이어 정체성 (설계 대원칙)
- 플레이어는 용사가 아니다. **용사 일행을 지켜보고 이끄는 별개의 존재**다.
- 마우스 = 그 존재의 시선이자 손. 게임의 조작 규칙은 단 두 문장으로 수렴한다:
  1. **몸은 부딪힌다 (범프)** — 파티의 모든 상호작용은 충돌로 일어난다.
  2. **시선에는 힘이 있다 (호버)** — 커서를 올리는 것 자체가 행동이다.
- 튜토리얼 없는 게임 = 규칙이 하나(우리는 둘)로 수렴하는 게임.

## 2. 세계관 — 비노출 설계 원리

### 2.1 플레이어에게 보이는 텍스트 (전부)
- **프롤로그 (3줄)**: 엄마 "일어나렴, 용사야." → 페이드 → **"용사는 일어났다. 그러나, 너무 늦었다."** → 필드 시작.
- **촌장 첫 대사**: "마왕은 이미 세계를 손에 넣었습니다. …그래도, 하시겠습니까?" (게임의 피치를 NPC가 친다. 튜토리얼 텍스트 대체.)
- **음유시인의 서사시**: 골드로 구매하는 수집형 스토리 조각. 내용 = "용사가 자는 동안 무슨 일이 있었는가" (마왕의 승리 과정, 이전 용사의 이야기). 플레이어가 돈 주고 사는 이야기가, 자신이 늦잠 자느라 놓친 이야기라는 구조.

### 2.2 우리끼리의 설정 (게임 내 절대 비노출)
- **늦잠**: JRPG 오프닝 클리셰(엄마가 깨움)의 반전. 최악은 이미 일어났으므로 더 나빠질 게 없다 → 플레이어가 항아리나 깨고 카지노를 돌려도 서사적 긴장이 깨지지 않는다. "부담 없는 게임"이라는 톤을 세계관이 승인한다. 또한 세계가 0으로 리셋된 상태에서 다시 쌓는 이야기 = 인크리멘탈 장르의 정의 그 자체.
- **양산박 / 환상수호전 기지**: 마을은 마왕 치하 최후의 자유 지대이자 저항군 기지. 건물 해금 = 흩어진 사람들이 소문을 듣고 합류하는 것. 동료 영입도 같은 문법 (저항자들의 합류, 꼬리물기 대열 = 저항군의 행진).
- **팔레트 = 지배자**: 마왕이 이미 이겼으므로 필드는 지역 지배자(사천왕 격)의 색으로 물들어 있다. 챕터 보스 = 그 지배자. 처치하면 필드 전체가 다음 지배자의 색으로 전환. "다른 대륙 이동"이 아니라 "같은 땅의 지배자 교체"이므로 단일맵이 설정상 필연이 된다. **마을만은 물들지 않는다** — 항상 따뜻한 색. 대사 없는 시각 언어.
- **멀티 전투창의 설득력**: 이미 진 전쟁을 뒤집으려면 정상 속도로는 안 된다. 파티는 모든 전선에서 동시에 싸운다. 훅과 서사가 같은 장면을 가리킨다.
- **시선의 정체**: 절대자가 아니라 **대등한, 이름 없는 또 하나의 동료**. 황금 슬라임을 잡아주고, 전투를 응원해주는 존재. **게임 내에서 일절 언급·암시하지 않는다.** 힌트 대사 금지. 침묵이 원칙.

### 2.3 늦잠 개그 훅 (텍스트/도전과제 재료)
- 여관 이용 시: "…늦잠은 금물." / 트링켓 "늦잠"(여관 회복 2배, 필드 복귀 지연) / 스팀 도전과제 "또 늦잠 잤다" 등. 늦잠 = 시그니처 개그.

## 3. 조작 체계 — 마우스 온리

### 3.1 손 사용 예산 (절대 원칙)
게임 전체의 손 사용은 아래 세 강도뿐이다. **연타는 금지.** 이 예산을 초과하는 신규 아이디어는 기각하거나 예산 내로 재설계한다.
1. **범프 (클릭)** — 이동/상호작용 명령. 클릭 1회 = 한동안의 행동.
2. **호버 (올려두기)** — 주시 버프, 설명창 열람. 클릭 없음.
3. **가끔 문지르기 (드래그)** — 황금 슬라임 포획, 대장간 리듬. 수 분에 1회 수준의 이벤트성.

### 3.2 클릭 = 깃발 / 지목
- **빈 땅 클릭 = 깃발**: 파티가 그 지점으로 행군, 도착 후 **주변을 자율 배회하며 알아서 범프** (몬스터 조우, 항아리 파괴 등). 디아블로식 클릭 연타 노동 방지의 핵심.
- **오브젝트/몬스터/건물/NPC 클릭 = 지목**: 파티가 걸어가서 그것을 범프. 상점 클릭 → 걸어가서 메뉴 열림. 이동과 상호작용이 클릭 하나로 통합.
- 경로상 몬스터와는 자연 충돌로 전투창 개방 → "잘 그은 동선 하나가 창 3개를 여는" 조작 숙련의 재미.
- 키보드는 편의만: 상점 열기, 게임 속도 토글 등 단축키. 필수 아님.

### 3.3 커서 4종 (어포던스 = 향수)
| 커서 | 대상 | 의미 |
|---|---|---|
| 발/부츠 | 빈 땅 | 클릭 = 행군 (모노키 아일랜드/루카스아츠 오마주) |
| 손가락 포인터 | 오브젝트·건물·NPC·몬스터 | 클릭 = 지목 |
| 눈 | 전투창 | 클릭 없음. 올려두는 것 자체가 행동 (주시 버프). "시선" 규칙의 물화 |
| 쓰다듬는 손 (+반짝이) | 황금 슬라임, 대장간 등 직접 접촉 대상 | 드래그 = 문지르기 |

- 커서에 은은한 금색 아우라. 상호작용 가능 대상에 접근하면 대상이 살짝 반짝인다 → "마우스를 대면 좋은 일이 생긴다"는 단일 문법이 게임 전체를 관통.
- 처음 보는 대상에 포인터를 올리면 설명창에 "?????" → 한 번 범프 후 정식 설명으로 갱신 (검시 = 도감 채우기).

### 3.4 상시 설명창
- 위치: 화면 하단, 드퀘식 파티 스테이터스 창들 **옆에 고정**. 팝업 툴팁이 아니라 상주 대화창 — 게임이 플레이어에게 계속 말을 거는 채널.
- 역할: 검시 텍스트, 시스템 튜토리얼 대체, 상태 변주 텍스트 (§14 톤 가이드 참조).
- 핵심 예시:
  - 전투창 호버: **"당신이 지켜보고 있다. 일행이 힘을 낸다!"** (호버 버프의 튜토리얼을 이 창이 흡수)
  - 황금 슬라임: **"쓰다듬어 주기를 바라는 것 같다."**
  - 항아리: "평범한 항아리다. 깨뜨리고 싶다."
  - 유령 동료: "전사의 유령. 억울해 보인다."

### 3.5 이동 자동화 아크 — "판단의 자율화"
자동이동(경로 추종)은 태생부터 패시브다. 수동→자동 아크는 한 층 위에서 일어난다:
1. **초반**: 목적지를 플레이어가 깃발로 정한다 (지휘관).
2. **중반 — "용사의 직감" 업그레이드**: 파티가 목적지를 스스로 정한다. 알아서 사냥터를 찾고, HP가 낮으면 알아서 여관에 들른다 (완전 자율 AI).
3. **이동 QoL 3단**: 이속 업그레이드 → **"루라"** (마을 즉시 귀환: 마을 아이콘 클릭/단축키, 중반 해금) → 파티 자율 귀환.

### 3.6 카메라 줌 = 정체성의 아크
- 마우스 휠 3단 정수 줌: **100%** (드퀘 스케일, 파티 밀착) / **50%** (2×2 스크린) / **33%** (월드 전체 한 화면).
- 초반 = 100%로 JRPG를 플레이하는 감각 → 후반 = 33%로 세계를 내려다보며 관장하는 감각. 수동→자동 아크가 카메라 거리로도 표현된다.
- **전투창은 UI 레이어이므로 줌과 무관하게 항상 같은 크기.** 줌아웃 시 필드는 작아지고 전투창들은 그대로 떠 있는 화면 = 대표 스크린샷/트레일러 마지막 컷.

## 4. 필드와 범프 (코어 루프)

### 4.1 범프 = 이 게임의 "긁기"
파티 선두가 무언가에 충돌하는 것이 유일한 상호작용 동사다.
- 몬스터에 쿵 → 전투창 팝업
- 항아리에 쿵 → 쨍그랑 + 골드 팝
- 보물상자에 쿵 → 벌컥
- NPC/건물에 쿵 → 대사/메뉴
- 반짝이(발굴 포인트)에 쿵 → 발굴 (삽 보유 시)
걷는 감각(발소리, 풀 흔들림, 꼬리물기 대열의 꼬물거림)에 juice를 집중 투자한다. 걷기 자체가 향수이자 촉감이다.

### 4.2 왕복 리듬 (게임의 심장박동)
**필드 출격 (창 굴리기·발굴) → HP 소모 → 마을 귀환 (여관 회복 · 교회 부활 · 항아리/상자 · 대장간 · 상점 쇼핑) → 재출격.**
- 마을 오브젝트(항아리·상자)가 "귀환할 이유"를, HP가 "귀환할 타이밍"을 만든다.
- 후반에는 자율 AI와 조수 동물이 이 순환을 대행하고, 플레이어는 관장한다.

### 4.3 필드 발굴 (HoMM 오마주)
- 필드 랜덤 위치에 반짝이는 발굴 포인트가 주기적으로 스폰.
- **삽** (상점 업그레이드) 보유 시 범프로 발굴 → 골드 팝, 낮은 확률로 훈장. 해금 전에는 반짝여도 못 판다 — 설명창: "뭔가 묻혀 있는 것 같다…" (약올리기 = 삽 구매 동기).
- 챕터가 진행되면 보상 티어 상승. 자동화는 조수 "꽃돼지" 담당 (§8).

### 4.4 재화 원칙
- **골드 하나.** 경험치는 자동 누적, 레벨은 알아서 오른다 (플레이어 관리 없음).
- 예외는 단 둘: 카지노 코인 (놀이용 칩, §7), 회차 포인트 (프레스티지, §12). 그 외 재화 신설 금지. 황금 슬라임 보상도 순수 골드.

## 5. 전투 시스템 — 멀티 전투창

### 5.1 전투창의 생애
1. 파티 선두가 필드 몬스터 1마리와 충돌 (인카운트).
2. 전투창 팝업. 창 안에는 **적이 여럿일 수 있다** ("슬라임이 3마리 나타났다!" — 조우 구성은 필드 몬스터의 티어/종류가 결정).
3. 창은 **자체 턴 타이머**로 자동 진행. 파티는 필드에서 계속 움직인다.
4. 승리 → 짧은 팡파레 + 골드/EXP가 창에서 필드로 물리적으로 튀어나옴 → 창 닫힘.
- 창 상한 도달 상태에서 몬스터와 충돌 시: 전투창이 열리지 않고 "전투창이 가득 찼다!" 피드백 (몬스터는 소모되지 않음).

### 5.2 파티 편재의 원리 (§1.2)
- 모든 창의 "카메라 뒤"에 파티 전원이 동시에 존재한다. 아군 스프라이트는 그리지 않는다 — 텍스트 로그("전사의 공격! 슬라임A에게 24의 데미지!")와 이펙트만으로 표현.
- **화력은 창 개수로 분산되지 않는다.** 전사의 검이 +5면 모든 창에서 +5 풀스윙. "다섯 전장 동시 풀파워"가 이 게임의 판타지다.
- 한 명의 사망도 모든 창에 동시 적용 (§6.4).

### 5.3 수입 공식과 비용
> **수입 = 동시 창 개수 (가로 확장) × 창당 적 수 (밀도) × 턴 속도 (회전율)**
- 세 축 모두 업그레이드/훈장의 대상.
- **비용은 전부 HP 하나로 수렴한다**: 각 창에서 적이 공격 턴을 가질 때마다 중앙 파티 HP(정확히는 타겟 동료의 HP)가 깎인다 (턴당 차감). "슬라임의 공격! 전사는 8의 데미지를 받았다!" 로그 + 창에서 하단 스테이터스 창으로 데미지 숫자가 날아가는 연출 (어디서 맞았는지 시각적으로 설득).
- 자연 브레이크: 창을 많이 열수록 여러 창에서 동시에 맞는다 → "상한만큼 다 여는 게 항상 최선이 아니라, 회복력이 감당하는 만큼 연다." 여관·승려·회복 업그레이드가 전부 유의미해지는 스로틀.
- 무리 창(적 5마리)은 벌이도 좋지만 맞는 것도 5배 — 밀도 업그레이드/훈장에 리스크 내장.

### 5.4 창별 턴 관리 (비동기)
- 창마다 독립 턴 타이머 → 창들이 제각각의 박자로 텍스트 갱신·이펙트·팡파레를 터뜨린다. 비동기가 화면의 "생명감"을 만든다. 후반 승리 팡파레가 어긋난 타이밍에 겹쳐 화음처럼 들리는 것이 시그니처 사운드.

### 5.5 호버 = 주시 (상시 개입층)
- 커서를 올려둔 창: **전투 속도 +50%, 크리티컬 +15%** (체감 기준치, 튜닝 대상).
- 피드백 (즉각적·가역적·첫 발생 시 시끄럽게 — 0.1초 내 온오프로 인과를 플레이어가 스스로 실험 가능해야 함):
  - 테두리 흰색 → 황금색
  - 로그 스크롤/이펙트 리듬이 눈에 띄게 빨라짐 (수치가 아니라 리듬의 변화)
  - 타격음 피치/템포 업
  - 설명창: "당신이 지켜보고 있다. 일행이 힘을 낸다!" / 커서 이탈 시: "…일행은 조금 외로워졌다."
- **호버는 끝까지 자동화하지 않는다.** 전부 자동화된 후반에도 플레이어 손의 마지막 가치로 남는다.

### 5.6 황금 슬라임 (이벤트 개입층 — 자동화가 못 먹는 잭팟)
- 열린 전투창 중 하나에 낮은 확률로 난입 (기준: 수 분당 1회 급). 등장 시 명확한 전용 효과음 (메탈 슬라임 "샤랑~" 오마주).
- **파티의 공격은 전부 회피** ("미스! 미스!" — 자동전투의 무능이 곧 규칙 튜토리얼). 수 초 후 도주.
- 포획 = **드래그 문지르기**: 커서 접근 시 슬라임이 커서를 쳐다보고 부들부들 떤다 → 닿으면 물컹 스쿼시 + "뽀잉" + 파티클 (눌리는 것을 본 인간은 문지른다) → 문지르면 원형 포획 게이지 상승, 스쿼시 격화 + 사운드 피치 계단 상승 → 완성 시 "황금 슬라임을 붙잡았다!" + 골드 분수 (대박 골드+EXP).
- 실패 시: "황금 슬라임은 도망쳐 버렸다!" + 창 밖으로 튀어나가 화면 밖으로 사라지는 연출 (놓친 것이 보여야 학습된다).
- **첫 등장은 스크립트**: 게임 시작 10~15분경 (창 1~2개 시기) 확정 등장 + 도주 타이머 2배. 이후 정상 확률/타이머.
- 관련 업글/훈장 훅: 수배 게시판 "황금 슬라임 목격 정보"(출현율 업), 훈장 "끈끈이 장갑"(도주 +2초), "슬라임 유인향"(호버 중인 창에만 등장 — 두 개입층을 잇는 훈장).

### 5.7 보스전
- 챕터 보스: 맵 외곽(동/서/남)에 챕터당 1개 출현. **유일하게 파티가 멈추고 전원이 붙는 대형 전투창.** 처치 시 팔레트 전환 + 신규 해금 (§10).
- 최종 보스(마왕): 북쪽 마왕성. 최종 챕터에만 문이 열린다. 처치 = 엔딩 → 프레스티지 (§12).

## 6. 파티 시스템 — 동료, 무기, 죽음, 유령

### 6.1 동료 (6~8명)
- 영입 = 꼬리물기 대열 +1 (시각 보상) + 전투 패시브 1개 (기능 보상). **동료는 전투와 전투 패시브에만 관여한다** (마을/필드 자동화는 조수 동물의 영역, §8).
- 전투 패시브 예시 (전원 전투창 내부 역할):
  - **용사**: 리더. 밸런스형.
  - **전사**: 피격 확률 높음 (탱커 = 어그로).
  - **마법사**: 전체 공격 (무리 창에 강함).
  - **승려**: 턴마다 파티 소량 회복 ("승려만은 지켜야 해" → 승려부터 죽었을 때의 절망까지가 콘텐츠).
  - **도적**: 처치 시 골드 훔치기 (골드 배율) + 선제공격 확률.
  - 이후 영입 동료도 같은 문법으로 (전투창 내부 역할 1개씩).
- **대열 순서는 플레이어가 변경 가능** (공짜 시스템 — 훈장 "후미의 긍지" 등이 순서를 빌드 요소로 만든다).

### 6.2 시그니처 무기 (장비 시스템의 전부)
> **"동료별 시그니처 무기 1개, 레벨만 존재. 인벤토리 없음, 파밍 없음, 교체 없음, 방어구/악세서리 없음."**
- 전사의 검, 마법사의 지팡이… 각 +1, +2, +3… 대장간에서 강화 (§7 대장간).
- 무기 레벨 = 해당 동료의 전투 성능 + 패시브 효율 동반 상승 → "어느 무기를 벼리냐 = 어떤 역할을 키우냐" = 장비가 곧 빌드.
- 역할 분화: 전사=단일 강타(정예 창), 마법사=전체기(무리 창) → 조우 분포에 따라 강화 우선순위 판단이 생긴다.
- **비용은 동료별 개별 스케일링** (몰빵하면 그 동료만 폭등 → 자연 분산 유도, 최적 단일 순서 고착 방지).
- 시각화: 레벨 구간별 무기 팔레트 스왑 (구리→은→금→발광) — 전투창 이펙트와 필드 대열 스프라이트에 반영. +5/+10 마디에서 **무기 진화**: "구리검이 강철검이 되었다!" (이름+도트 변경 = JRPG 장비 감성의 마일스톤 번역).
- **용사의 검만은 특별 취급**: 골드로 벼릴 수 없다. **서사시(음유시인) 해금과 연동**하여 성장 — "전설의 검은 돈으로 벼릴 수 없다." 스토리 구매에 기능적 무게 부여.

### 6.3 HP와 UI
- **동료별 개별 HP.** 화면 하단에 드퀘식 멤버별 미니 스테이터스 창 나란히 (이름+HP 숫자, 드퀘 폰트/테두리). 사망 시 해당 창 회색+유령 아이콘.
- 플레이어는 HP를 "관리"하지 않고 "관람"한다 — 회복은 여관/승려가, 결정은 "언제 돌아갈까" 하나.
- 각 창에서 적의 공격은 생존 멤버 중 랜덤 타겟, 단 전사는 피격 확률 가중.

### 6.4 죽음 — 유령 (마더2 오마주)
- HP 0 → 사망이 아니라 **유령 상태**: 대열에서 빠지지 않고 반투명 유령 도트로 계속 꼬리물기 (죽음 = 소멸이 아니라 상태 → 심리적 부담 최소화).
- 유령은 모든 전투창에서 전투 불가 + 패시브 정지. 여러 창에 "슬라임의 공격! 도적은 쓰러졌다!" 로그 동시 표기.
- 페널티는 그것으로 충분하다 (**전투가 느려진다 = 수입이 준다**가 내장 페널티). 추가 페널티 없음.
- **용사 사망 → 리더 교대**: 필드 선두 스프라이트가 대열의 살아있는 첫 동료로 교체. 유령 용사가 둥둥 따라온다. "우리 파티는 계속 간다."
- **부활 = 교회에서만**: 교회 범프 → 유령 전원 소생, 유령 1명당 소액 골드 (가벼운 싱크, "아이고 헌금" 감각). 여관은 HP 회복만 담당하며 부활 기능 없음.
- **전멸**: 화면 페이드 아웃 → **"눈앞이 깜깜해졌다…"** → 교회 앞에서 기상 + **소지금 20~30% 차감** (드퀘 정통의 자비 버전). 전멸 시에만 골드 페널티.
- 예상 곡선: 초반에는 죽음이 왕복 리듬을 가르치는 선생님, 중반 이후 자연 은퇴. 단 유령 빌드 훈장(§9)이 후반에 죽음을 재고용한다.

## 7. 마을 — 건물 8종 (건물마다 담당 시스템 1개, 중복 금지)

| 건물 | 담당 시스템 | 명세 |
|---|---|---|
| **항아리** (군집) | 단쿨 클릭 오브젝트 | 쿨타임마다 리스폰. 범프 → 골드 팝. 업글: 개수·보상, 티어 팔레트 스왑 (나무→도자기→황금). 자동화: 원숭이 |
| **보물상자** | 장쿨 클릭 오브젝트 + 리스크 | 쿨 길고 보상 큼. 업글로 **미믹 확률** 추가 — 미믹이면 전투창 개방, 승리 시 대박. 낮은 확률 훈장 드랍. 자동화: 상자지기 |
| **여관** | **HP 회복 전용** | 범프 → 파티 전원 HP 회복. 업글: 최대 HP / 회복량·속도. **부활·오프라인 기능 없음.** 텍스트: "…늦잠은 금물." |
| **상점** | 업그레이드 패널 (다이어제틱 UI) | 범프 → 구매 메뉴. 품목: 이속/전투창 상한/전투 속도/공격력/골드 배율/삽/루라/용사의 직감/자동화 계열. 업글: 품목 슬롯 확장. 후반 "상인의 텔레파시"로 원격 열람 |
| **교회** | 부활 + 프레스티지 + 전멸 기상 지점 | 유령 소생 (1명당 소액 골드). **"모험의 서에 기록하시겠습니까?"** = 회차 리셋 (§12) |
| **카지노** | 골드 싱크 + 분산(variance) | 코인 = 별도 재화 (골드로 구매, 환불 불가). 미니게임: **슬롯 1종으로 시작** (후반 "자동 슬롯" 업글, 회차 해금으로 신규 게임 추가). 교환소: 진행 필수급 1개 (전투창 상한 +1 등) + 한정 훈장 2~3종 + 치장템 → 카지노를 순례 코스로 |
| **대장간** | 유일한 액티브 미니게임 | 쿨타임마다 불이 켜짐 → 범프 → **동료 초상화 선택** ("이번엔 누굴 벼릴까") → 리듬 미니게임 (게이지 화살표 타이밍, 3~5회 두들김, 10초 컷) → 판정별 +1/+2/+3(회심의 필살작!). 미니게임은 게임 전체에서 이것 하나뿐. 후반 "제자 고용" = 자동 +1만 (필살작은 끝까지 손맛) |
| **음유시인** | 스토리 상점 + 도감 | 서사시 절 단위 골드 구매 → 짧은 회상 컷신. 마지막 절 = 최종 보스 입장 조건 (기능적 의미). 용사의 검 성장 연동. 회차마다 새 서사시 라인. 옆에 **훈장 도감** 전시 |

- 배치: 중앙 광장 + 8슬롯이 광장을 둘러쌈. 해금 전 슬롯은 빈 터/천막으로 표시 (빈 슬롯 = 광고판).
- 해금 연출: NPC 합류 → 짐 풀기 → 건물 건설 (§2.2 양산박).

## 8. 조수 동물 — 마을/필드 자동화 (동료와 분리)

> 원칙: **동료 = 전투 / 조수 = 자동화.** 죽음 페널티가 마을 경제를 침범하지 않고, 인과가 한눈에 읽힌다.

| 조수 | 담당 | 비고 |
|---|---|---|
| **원숭이** | 마을 항아리 자동 파괴 | 업글: 마릿수/속도. 티어 시각화 (1마리→3마리→황금 원숭이) |
| **상자지기** (쥐돌이 or 고블린 집사) | 보물상자 자동 개방 | ※ "도비"는 해리포터 고유명이므로 이름 변경 필수 (컨셉 유지) |
| **꽃돼지** | 필드 반짝이 자동 탐지+발굴 | 필드를 킁킁대며 배회. 업글: 마릿수/속도 |

- 조수는 3종으로 동결. 화면 위를 실제로 돌아다니는 "살아있는 업그레이드" (Gnorp의 생명감) — 마을 성장의 두 번째 축 (건물 + 주민).

## 9. 훈장 (배지) 시스템 — 빌드와 플레이어 표현

### 9.1 원칙
- **숫자를 올리는 훈장 금지. 규칙을 바꾸는 훈장만.** ("공격력 +10%"는 상점의 일이다.)
- 전 훈장 한 문장 설명 가능해야 함. 출시 기준 **20~25개**.
- **훈장 간 시너지 조합은 설계하지 않는다** (밸런싱 지옥 방지). 개별 훈장이 독립적으로 재미있으면 충분 — 조합 발견은 플레이어의 몫.

### 9.2 슬롯과 운영
- 슬롯 **3개 시작 → 회차 보상으로 최대 5~6개** (좁아야 선택이 아프다).
- 교체: 마을에서 언제든 무료 (실험을 벌주지 않는다).
- 획득: 회차를 넘어 **영구 소장**. 음유시인 옆 도감에 전시 (수집욕 + 스팀 도전과제 연동).

### 9.3 획득처 (신규 시스템 없이 기존 시스템에 분배)
- 보물상자 낮은 확률 드랍 / 챕터 보스 첫 처치 확정 / 카지노 교환소 한정 / 도전과제식 해금 (CloverPit 방식: "항아리 1000개 파괴", "미믹 10승" 등 플레이 습관이 해금 조건) / 회차 해금 풀 (2회차+).

### 9.4 훈장 예시 풀 (시스템당 1개 이상 비틀기)
- **미믹의 이빨**: 모든 상자가 미믹, 보상 3배
- **금 간 항아리**: 항아리 쿨 절반, 보상 절반
- **겁쟁이의 깃발**: 창 상한 −1, 남은 창 전투 속도 2배
- **왕수의 성수**: 받는 데미지 2배, 골드 2배
- **후미의 긍지**: 대열 맨 뒤 동료의 패시브 효율 2배 (대열 순서 = 빌드)
- **카지노 VIP 카드**: 슬롯 결과에 몬스터가 뜨면 필드에 실제 스폰
- **모루의 축복**: 대장간 실패 없음, 대신 +3도 없음
- **메탈 슬라임의 왕관**: 황금 슬라임 출현율 대폭 상승, 도주도 빨라짐
- **끈끈이 장갑**: 황금 슬라임 도주 +2초
- **슬라임 유인향**: 황금 슬라임이 호버 중인 창에만 등장
- **늦잠**: 여관 회복 2배, 필드 복귀 지연
- 유령 빌드 계열: **폴터가이스트** (유령이 필드 항아리를 통과하며 파괴), **원혼의 함성** (유령 1명당 생존 멤버 공격력 +15% — 일부러 죽이는 빌드), **심령 파티** (유령도 모든 창에서 절반 위력으로 공격), **순교자의 성표** (사망 순간 전 창에 폭발 데미지), **저승의 문지기** (유령 상태 동료의 강화 비용 반값)

## 10. 적 해금과 챕터 구조

### 10.1 이중 해금 (구매 + 자동)
- **챕터 내부 = 구매**: 촌장의 **현상수배 게시판**. 골드로 수배서를 떼면 (=정찰대 파견) 해당 몬스터가 필드 스폰 시작. 챕터당 3~4종 순차. "더 위험한 놈을 일부러 불러들인다" (저항군 설정 부합). 새 몬스터 = 더 아프지만 수입 티어 상승 (벽→돌파 리듬). 게시판에 "황금 슬라임 목격 정보" (출현율 업) 포함.
- **챕터 전환 = 자동**: 보스 처치 → 팔레트 전환과 함께 전 몬스터 자동 리스킨 + 스탯 배율. 새 챕터의 수배 게시판이 다시 채워진다.

### 10.2 거리 그라데이션
- 같은 챕터 안에서도 마을에서 멀수록 상위 몬스터/상위 발굴 스폰 (동심원 링). "멀리 갈수록 벌이가 좋은데 여관도 멀다"는 리스크 지형. 자율 AI 시기엔 "파티 행동반경" 업그레이드가 이 축을 승계.

### 10.3 챕터/팔레트 (4챕터 기준)
초원 (초록) → 황혼 (주황) → 설원 (하양·파랑) → 마계 (보라·검정). 몬스터 기본 도트 8~10종 × 팔레트 스왑 (드퀘 정통 기법 = 절약이 아니라 오마주). 챕터 보스는 동/서/남 외곽에 순차 출현, 마왕성 (북)은 최종 챕터에 개방.

## 11. 맵과 카메라 (수치 확정)

| 항목 | 값 | 근거 |
|---|---|---|
| 카메라 | **640×360** | 16:9 정수 스케일. 16×16 타일 기준 40×22.5타일 = 고전 JRPG 화면 밀도 |
| 타일 | **16×16** | 드퀘 패미컴 원전. 캐릭터 16×16, 건물 32×32~48×48 |
| 월드 | **1920×1080 px = 120×68타일 = 카메라의 정확히 3×3** | 정수 줌 성립 (100/50/33%), 픽셀아트 다운스케일 1/2·1/3 정수 |
| 파티 기본 이속 | **60~70 px/s** | 마을→맵 구석 편도 15~18초 ("출격" 감각은 있고 지루하기 전에 끝나는 길이, 이속 업글 체감 여백) |
| 마을 | 중앙, **약 30×20타일** | 중앙 광장 + 건물 8슬롯 환형 배치 |

### 구조
```
            [마왕성]          ← 북쪽 가장자리. 게임 시작부터 실루엣으로 상주
        외곽 링 (고티어)
          중간 링 (중티어)
            내곽 링 (저티어)
     ───────[마을]───────     ← 정중앙. 팔레트에 물들지 않는 유일한 지역
            내곽 링
          중간 링
        외곽 링
```
- **십자 가로 4개** (동서남북): 길 위 몬스터 스폰 없음 → "안전 고속 이동 vs 벌판 범프 파밍" 동선 선택 + 패스파인딩 가독성.
- 팔레트 전환은 동일 지오메트리 위에서 색만 교체 — **맵은 하나만 만든다.** 챕터별로 스폰/반짝이/오브젝트 밀도만 조정.
- 맵 확장 금지. 인크리멘탈의 콘텐츠는 공간이 아니라 수의 성장과 시스템 해금이다. 맵이 크면 밀도가 희석된다.

## 12. 프레스티지 — 회차 (New Game+)

- 마왕 처치 → 엔딩 → 교회 "모험의 서에 기록하시겠습니까?" → 회차 리셋.
- 리셋 시 **회차 포인트** 획득 → 영구 강화 트리 (시작 전투창 수, 이속, 초기 동료, 훈장 슬롯 +1 등).
- **영구 유지**: 훈장 컬렉션, 도감, 서사시 구매 이력(열람).
- **회차 해금 콘텐츠** (리텐션 엔진 — "다음 회차엔 뭐가 풀리지"): 신규 훈장 풀, 카지노 신규 미니게임, 신규 서사시 라인 ("2회차에서만 들을 수 있는 진실"), 기타 스코프 동결로 미뤄둔 아이디어의 수납처.
- 회차는 훈장과 결합해 "이번 회차는 이 빌드로" (발라트로식)가 되어야 한다 — 같은 게임 반복이 아니라 빌드 반복.

## 13. 경제 / 밸런스 가이드

- **수입 3축**: 창 개수 × 창당 적 수 × 턴 속도. **비용 단일 수렴**: 전부 HP (턴당 차감) → 회복 계열이 스로틀.
- **골드 싱크**: 상점 업그레이드 (지수 스케일), 무기 강화 (동료별 개별 스케일), 수배서, 서사시, 카지노 코인, 교회 헌금 (유령 부활), 전멸 페널티 (20~30%).
- **분산 담당**: 카지노 (한 방), 황금 슬라임 (액티브 잭팟), 미믹 (리스크), 발굴 (소소한 랜덤).
- **액티브 프리미엄**: 호버 (+50% 속도/+15% 크리), 황금 슬라임, 대장간 +3 — 방치해도 손해가 아니라 액티브만 이득인 구조 유지.
- **오프라인 없음**의 보완: 껐다 켰을 때 **3초 내 재미 복귀** (자동 저장, 즉시 필드 로드, 복귀 브리핑 없음).
- 벽(진행 정체)의 해소는 항상 다음 중 하나로: 수배서 해금 / 챕터 보스 도전 / 훈장 교체 / 회차. 벽이 "기다림"으로만 풀리는 구간을 만들지 않는다.
- 수치 초안은 스프레드시트로 별도 관리 권장. 본 문서의 수치는 체감 기준선.

## 14. 플레이버 텍스트 톤 가이드

- **화자**: 드퀘식 3인칭 내레이터. 건조한 서술 속 유머 ("~것 같다", "~하고 싶다").
- **언더테일 한 스푼**: 대상의 감정을 태연하게 서술 — "용사가 힘을 낸다.", "슬라임은 부끄러워한다.", "쓰다듬어 주기를 바라는 것 같다."
- **상태 변주**: 같은 대상도 상황 따라 텍스트가 바뀐다 (챕터 팔레트: "얼어붙은 슬라임. 조금 딱딱해 보인다." / 전멸 직후 촌장: "…괜찮습니다. 다들 그렇게 시작합니다."). 텍스트 수백 줄 = 가장 싼 인격 부여 콘텐츠.
- **금지**: 시선의 정체에 대한 언급·암시 일절 금지 (§2.2). 제4의 벽 관련 텍스트는 넣지 않는다.
- **드퀘 관용구 활용**: "○○이(가) 나타났다!", "회심의 일격!", "눈앞이 깜깜해졌다…", "전투창이 가득 찼다!"
- 대사는 항상 짧게. 긴 텍스트는 컨셉("시간 없는 현대인용 JRPG")과 자기모순.

## 15. 개발 로드맵 (우선순위 순)

### Phase 1 — 재미 검증 프로토타입
클릭 이동 (깃발/지목) + 꼬리물기 대열 + 범프 + **전투창 1개** (자체 턴, 다수 적, 로그, 승리 골드 팝) + 항아리.
> **게이트: 이 상태로 10분이 재미있는가?** 여기서 재미없으면 건물을 백날 지어도 소용없다. 걷기/범프/팡파레 juice를 이 단계에서 완성한다.

### Phase 2 — 인크리멘탈 성립
멀티 전투창 (상한 업글) + 상점 (핵심 업글 라인) + HP/여관 왕복 리듬 + 호버 주시 + 커서 4종 + 설명창.

### Phase 3 — 루프 완성 (수직 절편)
동료 2~3명 + 유령/교회 + 수배 게시판 + 첫 챕터 보스 + 팔레트 전환 1회 + 프레스티지 골격. → **데모 빌드 분기점** (첫 동료 + 창 2개 + 첫 보스, 30~40분).

### Phase 4 — 살 붙이기
카지노 + 대장간 + 음유시인 + 조수 3종 + 발굴/삽 + 황금 슬라임 + 훈장 20~25종 + 루라 + 잔여 동료/챕터.

### Phase 5 — 마감
회차 콘텐츠, 밸런스 패스, 도전과제, 로컬라이즈 (한/영 우선), Steam 페이지/트레일러 (마지막 컷 = 33% 줌 + 전투창 도배).

## 16. Godot 4.7 기술 노트 (Claude Code 착수 가이드)

### 16.1 프로젝트 설정
- 뷰포트 640×360, Stretch Mode `canvas_items` + `integer` scale (도트 무결성).
- 픽셀 스냅: `snap_2d_transforms_to_pixel`, `snap_2d_vertices_to_pixel` 활성화. 텍스처 필터 Nearest.
- 커서: `Input.set_custom_mouse_cursor()` 로 4종 스왑 (발/포인터/눈/손). 호버 판정은 Area2D/Control의 `mouse_entered/exited`.

### 16.2 아키텍처 — 전투 로직과 렌더링 분리
- **전투 시뮬레이션은 순수 데이터 클래스** (RefCounted, 노드 아님): 파티 스탯 참조 + 적 구성 + 턴 타이머 tick. 렌더링과 완전 분리.
  - 이유: 후반 창 8개+ 동시 가동 시 성능, 줌아웃/최소화 상태에서 로직만 굴리기, 밸런스 헤드리스 테스트 가능.
- **BattleWindow.tscn** = 시뮬 1개를 구독해 그리는 뷰. 인스턴싱으로 멀티 창 구현. 각 창은 자기 시뮬의 시그널(턴 진행/피격/승리)만 듣는다.
- 파티 스탯은 싱글턴(Autoload) `PartyState` 하나로 — 모든 창이 같은 상태를 참조 (편재의 원리 = 구현상으로도 단일 소스).

### 16.3 레이어 구조
```
Main
├─ World (Node2D)            # 줌 대상
│   ├─ TileMapLayer          # 단일 맵, 팔레트 셰이더
│   ├─ Village / FieldObjects (항아리·상자·반짝이·성소)
│   ├─ Party (선두 + 꼬리물기 Follower들, 유령 포함)
│   ├─ Monsters / Assistants (원숭이·상자지기·꽃돼지)
├─ UILayer (CanvasLayer)     # 줌 무관 고정 크기
│   ├─ BattleWindows (컨테이너, 자동 배치)
│   ├─ PartyStatusBar (드퀘식 멤버 창들)
│   ├─ DescriptionBox (상시 설명창)
│   └─ Menus (상점 등)
```
- **전투창은 반드시 CanvasLayer** — 줌아웃 시 필드는 축소, 창은 고정 (대표 스크린샷의 기술적 근거).
- 카메라: Camera2D, zoom 1.0 / 2.0 / 3.0 역수 단계 (휠 입력), 트윈으로 스냅.

### 16.4 팔레트 스왑
- canvas_item 셰이더 + 팔레트 LUT 텍스처 (원본 인덱스 색 → 챕터별 램프 교체). 필드 타일맵과 몬스터 스프라이트 공용. 챕터 전환 시 LUT 보간으로 "세계가 물드는" 연출.

### 16.5 이동/AI
- 클릭 이동: 십자 가로+개활지 구조이므로 NavigationRegion2D 또는 단순 그리드 A* 중 가벼운 쪽. 꼬리물기 = 선두의 위치 히스토리를 간격 두고 추종 (드퀘 대열 방식).
- 자율 AI ("용사의 직감"): 상태기계 — 사냥터 선택(행동반경 내 최고 티어) → 배회/범프 → HP 임계 시 여관 → 반복. 유령/교회 경유 포함.

### 16.6 데이터 주도 설계
- 몬스터/업그레이드/훈장/수배서/서사시/플레이버 텍스트 전부 Resource(.tres) 또는 JSON 테이블로. 훈장은 "효과 훅 enum + 파라미터" 구조로 통일해 20~25종을 데이터로만 추가 가능하게.
- 세이브: 자동 저장 (주기 + 주요 이벤트), JSON 직렬화, Steam Cloud 대응. 오프라인 시뮬레이션 코드 없음 (설계상 불필요 — 이득).

### 16.7 사운드
- 창별 개별 짧은 팡파레 (비동기 겹침 허용 — 화음이 시그니처). 호버 시 타격음 피치 시프트 (`pitch_scale`). 황금 슬라임 전용 등장음.
